# 问道 · 更新日志
## [v0.4.2] - 2026-05-03
### 新增
- 装备栏系统：背包右侧改为像素人形装备栏，6个装备槽位（头盔/护甲/护腿/腰带/武器/护符）
- 像素风格成年男性轮廓（19个矩形块拼接），#00ffcc青色填充+边框
- 装备槽50×50px分布在人形周围，虚线连接到对应身体部位，空槽显示图标文字
- 道具详情改为点击弹窗显示，支持自动避免超出屏幕边界
- 法宝/功法类道具支持装备到对应槽位（法宝→武器槽/护符槽，功法→护符槽）
- 装备槽位支持换装（旧装备自动返回背包，背包满时拒绝换装并提示）
- 点击已装备槽位弹出详情弹窗，支持卸下（卸下时背包满则拒绝）
- 存档新增 equipment 字段记录当前装备状态
- 修炼方向系统：三大方向（体修/剑修/神识），每突破一小层获得3点修炼点
- 修炼方向影响战斗属性（体修→血量/近战/防御，剑修→灵力/剑气/回蓝，神识→暴击率/暴伤/移速）
- 战斗行为倾向系统：近战累积体修倾向，剑气累积剑修倾向，暴击累积神识倾向
- 倾向折合有效点数（每5点倾向=1点有效，上限不超过分配点数的50%）
- 修炼大厅重构：移除灵石升级属性按钮，改为三大修炼方向分配面板+战斗属性预览
- 暴击机制：暴击率/暴伤加成，暴击伤害橙色数字显示
- 防御机制：受击减伤，最少承伤1点
- 永久强化天材地宝改为奖励修炼点（permHp/permAtk/permMp→1点，permAll→2点）
### 改动
- 背包右侧面板从"道具预览"改为"装备栏"（像素人形+6槽位虚线连接布局）
- 装备槽从头盔/护符/武器/法宝/护甲/鞋履调整为头盔/护甲/护腿/腰带/武器/护符
- 道具交互从点击右侧预览改为点击格子弹出独立详情小窗
- SaveManager.getDefaultSave/createNew 新增 equipment 初始字段（新槽位key）
- 移除修炼大厅"淬体强筋/炼气凝神/拓展灵海"三个灵石升级按钮
- GameScene 属性计算从 save.upgrades 迁移到 calcPlayerStats(save)
- 存档新增 cultivation 字段（points + 三方向分配点数 + tendencies）
- helpers.js 新增 calcPlayerStats 函数，checkBreakthrough 突破时奖励修炼点
- MP回复从固定5%改为剑修修炼方向决定（基础5/s + 剑修加成）
- 移动速度改为神识修炼方向决定（基础220 + 神识加成）

## [v0.4.0] - 2026-05-03
### 新增
- 多存档系统：支持4个独立存档槽位，槽位数据单独存储
- 存档选择界面（SaveSelectScene）：显示角色名、境界、游玩时长、最后保存时间
- 新建存档时可自定义角色道号（最多8字），Phaser DOM输入弹窗
- 自动保存：所有主要场景每30秒自动保存，右下角显示"✦ 已自动保存"提示
- 游玩时长追踪：GameScene中累计游玩时间，自动保存时写入存档
- SaveManager工具类：统一管理所有存档操作（load/save/delete/格式化）
- 兼容旧版单存档（localStorage['wudao_save']）自动迁移到槽位0
### 改动
- 版本号统一由 config.js 的 GAME_VERSION 管理，不再硬编码
- 主菜单重构为右侧三按钮布局（开始游戏/存档/设置），纯文字按钮+左侧竖线装饰+悬停动画
- 存档界面支持 play 和 manage 两种模式：play 正常进入游戏，manage 查看详情和删除
- 存档数据通过 Phaser registry 全局传递，不再各场景单独读写 localStorage
- 游戏启动流程：BootScene → SaveSelectScene → MenuScene（新增存档选择环节）
- helpers.js 移除 loadSave/saveSave，checkBreakthrough 增加 slotId 参数
- 所有场景导入从 {loadSave,saveSave} 迁移到 registry + SaveManager
- HallScene / CaveScene / MeditateScene / TetrisScene / GachaScene / BagScene 均加入自动保存
